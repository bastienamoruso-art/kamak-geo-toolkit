Comment lire ce rapport

Les 4 sections du rapport

  • Vue d'ensemble — taux de mention de votre marque par provider, en pourcentage. C'est votre score de présence dans le modèle (ou dans le web search si --web-search est activé).
  • Concurrents cités — qui le modèle nomme spontanément quand on lui pose la question persona. Ce sont les marques que le modèle considère comme références sur ce marché.
  • Sources web citées — URLs et domaines réellement utilisés par le modèle (uniquement si --web-search est ON, sinon vide ou hallucinable). Ce sont les sites où placer votre marque pour entrer dans le corpus retrieval.
  • Détail prompt par prompt — chaque appel API avec sa réponse, filtrable par provider.

Comment interpréter votre taux de mention

  • 0 % — votre marque n'est pas dans le corpus du modèle (mode sans search) ou aucune source web ne vous référence sur ces requêtes (mode search). Action : pression de marque (PR, citations tierces) ou SEO local sur les requêtes manquées.
  • 10-30 % — vous êtes connu mais pas dominant. Vous êtes cité sur les prompts les plus génériques, pas sur les recommandations précises. Action : travailler les comparatifs/listicles sectoriels où vous figurez avec vos concurrents.
  • 40-60 % — vous êtes une référence reconnue sur le marché. Action : tenir cette position et capitaliser sur les prompts FAQ/recommandation où vous n'êtes pas encore cité.
  • 60 %+ — vous êtes top of mind pour ce modèle. C'est rare et précieux. Action : monitorer mois après mois pour ne pas perdre la position.

Web search ON vs OFF — interprétation différente

Mode web search OFF (par défaut sans --web-search) : vous mesurez la présence de votre marque dans le corpus d'entraînement du modèle. Si vous n'y êtes pas, vous êtes invisible sur l'interface standard de ces assistants quand l'utilisateur n'active pas le mode web.

Mode web search ON (--web-search) : vous mesurez ce qu'un utilisateur reçoit quand il active le mode "rechercher sur le web" côté UI. C'est plus proche de l'usage réel en 2026, et plus dépendant de votre SEO classique.

Un site bien référencé en Google peut être visible en mode search mais invisible en mode sans search. C'est précisément ce que ce rapport vous permet de mesurer.

Différences entre providers — c'est attendu

  • OpenAI (gpt-4o-mini) — très conservateur sans web search, refuse souvent de citer des marques. Avec search, ramène les sources de son moteur (Bing principalement).
  • Anthropic (Claude Sonnet 4.5) — plus enclin à citer des marques même sans search. Citations structurées avec annotations.
  • Perplexity (sonar) — toujours en mode search natif. Ce qu'il retourne est ce qu'un utilisateur voit dans l'app Perplexity. Le plus proche de la vérité prospect.
  • Gemini (2.5 flash) — sans search, basé sur le corpus Google. Avec search (google_search), ramène les résultats Google indexés.

Un écart de mention entre 2 providers n'est pas un bug — c'est un signal sur quels corpus/index vous référencent.

Limites à connaître

  • Variance — les réponses LLM ne sont pas déterministes. Refaire tourner le même prompt peut donner un résultat légèrement différent. Comptez sur la tendance sur plusieurs runs, pas sur une mesure unique.
  • L'API n'est pas l'interface réelle — pas d'historique utilisateur, pas de géolocalisation, pas de routing dynamique entre versions du modèle. Croisez avec le test manuel dans prompts/ pour la vérité prospect complète.
  • Web search hallucinable — sans --web-search, si une URL apparaît dans le texte de réponse, elle peut être inventée. Avec --web-search, les URLs sont des vraies pages crawlées par le provider.
